## 구현 로드맵 (Firebase 기반)

## Phase 0: 기초 셋업 (현재)
- `Next.js + TypeScript + Firebase` 초기 프로젝트 뼈대 생성
- 공통 타입/유틸(레인 계산, 점수 집계) 정리
- PRD, 아키텍처, Firestore 스키마 문서화

## Phase 1: 인증/권한 (우선순위 1)
- 관리자 로그인(Firebase Auth) 도입
- 규칙 기반 읽기/쓰기 정책 분리
  - 공개 페이지: 로그인 불필요 조회
  - 관리자 페이지: 인증 사용자만 관리 기능 허용

## Phase 2: 대회 관리 API
- 대회 CRUD
- 종별/세부종목 CRUD
- 대회 기간·레인 범위 설정

## Phase 3: 선수관리
- 선수 등록/수정/삭제
- 선수번호 자동 증가 규칙 구현
- 선수 조회/검색/필터 (소속, 시도, 조)

## Phase 4: 배정 엔진
- 랜덤 배정 API
- 게임별 배정 보드 생성 API
- 드래그앤드롭 수동 교체 API(교체/취소/복구)

## Phase 5: 점수/순위
- 게임 총점 입력 API
- 이벤트별 순위 계산 함수(합계/평균/핀차이)
- 전체종합순위 계산 함수(이벤트 합산)
- 실시간 리스너 반영

## Phase 6: UI 구현
- 공개 페이지
  - 대회 카드 대시보드(검색/필터)
  - 대회 상세 + 성적표/종합성적
- 관리자 페이지
  - 대회/종별/세부종목 편집 화면
  - 선수 등록, 랜덤 배정, 점수 입력/레이스 보드

## Phase 7: 운영 강건성
- 감사로그(관리자 조작 기록)
- 오프라인 대응(재시도 큐)
- 엑셀 export
- 성능 튜닝 및 인덱스 적용

## 8주 예시 일정
1~2주: Phase 0~2  
3주: Phase 3  
4~5주: Phase 4~5  
6주: Phase 6 MVP 릴리즈  
7~8주: Phase 7 운영 안정화
