# 5인조 전후반 연동 단순화 Design

## 목표

`세부종목관리`에서 5인조 전반/후반을 사람이 직접 서로 연결하는 방식을 없애고, 후반전을 만들 때 전반전을 선택한 뒤 "전반전 데이터와 연동하시겠습니까?" 확인을 통해 필요한 데이터를 한 번에 복사한다.

## 현재 문제

- 관리 화면에서 전반/후반과 연결 이벤트를 직접 맞춰야 해서 운영자가 어떤 값을 넣어야 하는지 직관적이지 않다.
- 실제 운영에 필요한 것은 "후반전은 전반전 기반으로 시작한다"는 초기화 흐름인데, 현재 UI는 그 의도를 잘 드러내지 못한다.
- 5인조는 `출전선수등록`, `스쿼드`, `팀 편성`, `로스터/선수교체`가 함께 움직이므로 단순히 이벤트만 연결해서는 운영 흐름이 완성되지 않는다.

## 승인된 동작

- 5인조 후반전 생성 시 운영자는 기존 전반전 하나만 고른다.
- 후반전 생성 직후 "전반전 데이터와 연동하시겠습니까?" 확인창을 보여 준다.
- 확인 시 전반전의 `출전선수등록(participants)`, `스쿼드(squads)`, `팀 편성(teams)`, `레인 배정(assignments)`을 후반전으로 복사한다.
- 이 복사는 생성 시 1회만 수행한다.
- 이후 전반/후반 데이터는 서로 자동 동기화하지 않는다.
- 다만 `linkedEventId`와 `halfType`는 유지하여 5인조 전반+후반 팀 종합점수 계산은 계속 동작해야 한다.
- 후반전 팀 기록은 후반전 시점의 `memberIds`를 기준으로 계산한다. 즉 전반전 이후 선수교체가 있었다면 교체 아웃 선수 대신 교체 인 선수의 후반전 기록이 팀 점수에 반영되어야 한다.

## 설계 방향

### 1. 세부종목 등록 UI 단순화

- 5인조 설정 UI에서 기존 `연결된 이벤트` 드롭다운을 제거한다.
- 대신 `전반/후반 구분`과, 후반전일 때만 보이는 `연동할 전반전` 선택 UI를 둔다.
- 생성 시에는 선택한 전반전의 ID를 프런트에서만 사용하고, 서버 저장 시에는 기존 필드인 `linkedEventId`로 저장한다.
- 수정 화면에서는 기존처럼 `halfType`과 `linkedEventId`를 보여 주되, 의미는 "종합점수 계산에 쓰이는 연결 정보"로 제한한다.

### 2. 후반전 생성 직후 1회 복사

- 후반전 생성 성공 후에만 확인창을 띄운다.
- 사용자가 확인하면 별도 API를 호출해 전반전의 `participants`, `squads`, `teams`를 후반전으로 복사한다.
- 취소하면 이벤트 연결 정보만 저장하고 데이터 복사는 하지 않는다.

### 3. 복사 범위와 규칙

- `participants`: 각 선수의 `playerId`, `squadId`를 그대로 복사한다.
- `squads`: 후반전에서 새 문서 ID를 만들고, 전반전 참가자의 `squadId`를 새 ID 기준으로 매핑해 함께 변환한다.
- `teams`: `memberIds`, `rosterIds`, `name`, `teamType`를 복사한다.`r`n- `assignments`: 전반전 마지막 게임 레인을 기준으로 후반전 `tableShift`를 1회 적용해 후반전 1게임을 만들고, 이후 게임은 후반전 `tableShift` 규칙으로 전개한다. 마지막 게임에 배정되지 않은 선수는 후반전에서도 미배정으로 둔다.
- 팀 간 연동용 `linkedTeamId`는 복사 과정에서 새로 계산하지 않는다. 이번 목적은 후반전 시작 상태 초기화이지 전후반 팀 문서의 실시간 연결 유지가 아니기 때문이다.

### 4. 선수교체 이후 점수 의미

- 전반전 복사는 초기 상태만 맞춘다.
- 후반전에서 선수교체가 일어나면 후반전 팀 문서의 `memberIds`가 바뀌고, 현재 점수 계산 로직은 각 이벤트의 팀 문서 기준으로 집계하므로 후반전 기록은 교체 인 선수 기준으로 자연스럽게 계산된다.
- 따라서 종합점수는 "전반은 전반전 팀 구성, 후반은 후반전 팀 구성"으로 합산된다.

## 영향 범위

- 관리자 이벤트 등록 화면
- 5인조 후반전 생성 플로우
- 이벤트 하위 데이터 복사 API
- 5인조 팀 생성 시 연결 이벤트에 자동 복제하던 로직 정리 여부 검토

## 검증 포인트

- 후반전 생성 후 확인창에서 확인을 누르면 출전선수, 스쿼드, 팀이 복사된다.
- 확인창에서 취소하면 후반전 이벤트만 생성되고 운영 데이터는 비어 있다.
- 후반전에서 선수교체 후 팀 점수와 전후반 합산 점수가 교체 인 선수 기준으로 계산된다.
- 기존 전반+후반 합산 리더보드가 유지된다.
